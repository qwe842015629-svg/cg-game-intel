# 客服页面管理模块 - 快速测试指南

## 🎯 测试目标

验证客服页面管理模块的前后端功能是否正常工作。

## 📋 测试步骤

### 1. 验证后台管理（5分钟）

#### 1.1 访问后台
```
URL: http://127.0.0.1:8000/admin/
登录: 使用管理员账号
```

#### 1.2 检查菜单
- 在左侧菜单中找到 **"客服管理"** 模块
- 应该包含3个子菜单：
  - ✅ 联系方式
  - ✅ 常见问题  
  - ✅ 页面配置

#### 1.3 测试联系方式管理
```
路径: 客服管理 > 联系方式
```

**操作**:
1. 查看列表，应该显示4条记录
2. 点击任意一条进行编辑
3. 修改"描述"字段
4. 保存并查看是否成功

**预期结果**:
- 列表显示：联系方式类型、标题、联系信息、启用状态、排序
- 可以直接在列表页修改"是否启用"和"排序"字段
- 保存后立即生效

#### 1.4 测试常见问题管理
```
路径: 客服管理 > 常见问题
```

**操作**:
1. 点击"增加常见问题"
2. 填写：
   - 问题：测试问题
   - 回答：测试回答
   - 分类：测试分类
3. 保存

**预期结果**:
- 成功创建新的FAQ
- 列表中显示新增的问题
- 查看次数初始为0

#### 1.5 测试页面配置
```
路径: 客服管理 > 页面配置
```

**操作**:
1. 点击唯一的配置记录进行编辑
2. 修改页面标题为："在线客服中心"
3. 修改页面描述
4. 保存

**预期结果**:
- 只有一条配置记录
- 无法添加新的配置（按钮不可见）
- 无法删除配置

### 3. 验证前端页面（5分钟）

#### 3.1 访问客服页面
```
URL: http://localhost:5178/customer-service
```

#### 3.2 检查页面内容

**应该看到**:
- ✅ 页面标题：显示后台配置的标题（如"在线客服中心"）
- ✅ 页面描述：显示后台配置的描述
- ✅ 联系方式卡片：4个（在线客服、邮件、电话、微信）
- ✅ 常见问题列表：4-5个问题

#### 3.3 检查浏览器控制台

**打开开发者工具 (F12)**，在Console标签应该看到：
```
成功加载客服页面配置: {id: 1, page_title: "在线客服中心", ...}
成功加载联系方式: 4 个
成功加载常见问题: 4 个
```

#### 3.4 测试动态更新

**操作步骤**:
1. 回到后台管理
2. 禁用一个联系方式（取消"是否启用"勾选）
3. 保存
4. 回到前端页面
5. 按 **Ctrl + Shift + R** 强制刷新

**预期结果**:
- 前端只显示3个联系方式卡片
- 被禁用的联系方式不再显示

### 4. 验证数据同步（3分钟）

#### 4.1 修改后台数据

```

#### 4.2 验证前端更新

```

**操作**:
1. 刷新页面（Ctrl + Shift + R）
2. 查看"充值需要多长时间？"的回答

**预期结果**:
- 显示修改后的内容："充值通常在5分钟内到账"

#### 4.3 修改页面配置
```
路径: 客服管理 > 页面配置
```

**操作**:
1. 取消"显示联系方式"勾选
2. 保存
3. 刷新前端页面

**预期结果**:
- 联系方式卡片全部消失
- 只显示常见问题列表

## 🐛 常见问题

### 问题1: 前端显示不更新
**解决方案**: 
1. 清除浏览器缓存（Ctrl + Shift + Delete）
2. 强制刷新页面（Ctrl + Shift + R）
3. 检查浏览器控制台是否有错误

### 问题2: API返回404
**解决方案**:
1. 检查Django服务器是否运行
2. 检查URL路由配置是否正确
3. 确认customer_service应用已添加到INSTALLED_APPS

### 问题3: 后台菜单不显示
**解决方案**:
1. 检查settings.py中的SIMPLEUI_CONFIG配置
2. 重启Django服务器
3. 清除浏览器缓存后重新登录

### 问题4: 数据库迁移失败
**解决方案**:
```bash
# 删除迁移文件重新创建
rm customer_service/migrations/0001_initial.py
python manage.py makemigrations customer_service
python manage.py migrate customer_service
```
