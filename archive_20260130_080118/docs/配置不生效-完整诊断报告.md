# 🔍 配置不生效 - 完整诊断报告

## 📋 问题现象

**后台配置：**
- 热门游戏：`display_count: 8`
- 最新资讯：`display_count: 6`

**前端实际显示：**
- 热门游戏：**5 个**
- 最新资讯：**4 个**

---

## 🎯 根本原因分析

### 问题1：热门游戏数据源限制

**数据源：** `frontend/src/data/games.ts`

**问题代码：**
```typescript
export const rechargeGames: RechargeGame[] = [
  { id: '1', name: '原神', hot: true, ... },      // ✅
  { id: '2', name: '王者荣耀', hot: true, ... },  // ✅
  { id: '3', name: 'Mobile Legends', hot: true, ... }, // ✅
  { id: '4', name: '和平精英', hot: true, ... },  // ✅
  { id: '5', name: 'Free Fire', hot: true, ... }, // ✅
  { id: '6', name: 'PUBG Mobile', hot: false, ... }, // ❌ 不是热门
]
```

**分析：**
1. 数组中只有 **5 个游戏** 标记为 `hot: true`
2. 即使后台配置 `display_count: 8`，也只能显示 5 个
3. 因为 `.filter(g => g.hot)` 只能过滤出 5 个游戏

**解决方案：**
需要在 `games.ts` 中再添加至少 3 个 `hot: true` 的游戏。

---

### 问题2：资讯API返回数量限制

**API调用：** `getSmartRecommendedArticles(displayCount)`

**可能的问题：**
1. 后端数据库中只有 4 篇文章
2. 后端API有硬编码的限制
3. 推荐算法过滤后只剩 4 篇

**验证方法：**
访问 API：`http://127.0.0.1:8000/api/articles/articles/smart_recommended/?limit=6`

## 🎯 快速修复脚本

将以下代码保存为 `fix-display-count.js`，然后在浏览器控制台运行：

```javascript
// 强制重新加载配置并应用
async function fixDisplayCount() {
  console.log('🔄 正在重新加载配置...')
  
  // 1. 清除本地缓存
  localStorage.clear()
  sessionStorage.clear()
  
  // 2. 重新加载布局配置
  const layouts = await fetch('http://127.0.0.1:8000/api/layouts/').then(r => r.json())
  
  // 3. 显示配置信息
  layouts.forEach(layout => {
    console.log(`📌 ${layout.sectionName}:`)
    console.log(`   - display_count: ${layout.config.display_count}`)
    console.log(`   - title: ${layout.config.title}`)
  })
  
  // 4. 强制刷新页面
  console.log('✅ 配置已更新，即将刷新页面...')
  setTimeout(() => {
    location.reload(true)
  }, 2000)
}

fixDisplayCount()
```
