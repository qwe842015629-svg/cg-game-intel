# ✅ 配置功能分离 - 完成报告

## 📌 问题描述

**用户反馈：**
> "请将功能分开，我现在看下来，许多功能的可视化配置都杂糅在一起了"

**问题分析：**
- 所有板块的配置字段都显示在同一个表单中
- 编辑轮播图板块时，也看到游戏列表、资讯、分类等无关字段
- 配置混乱，容易误操作
- 不知道哪些字段是当前板块需要的

---

## ✅ 解决方案

### 核心思路：为每个板块类型创建专属表单

通过Django Admin的`get_form()`方法，根据板块的`section_key`动态选择对应的表单类，实现：
- 编辑轮播图 → 只显示轮播专属配置
- 编辑热门游戏 → 只显示游戏列表配置
- 编辑资讯 → 只显示资讯专属配置
- 以此类推...

## 🛠️ 技术实现

### 核心代码：动态表单选择

在 `main/admin.py` 中：

```python
from .forms import (
    BannerSectionForm,
    BaseConfigForm,
    HotGamesSectionForm,
    LatestNewsSectionForm,
    FeaturesSectionForm,
    CategoriesSectionForm,
)

@admin.register(HomeLayout)
class HomeLayoutAdmin(admin.ModelAdmin):
    
    def get_form(self, request, obj=None, **kwargs):
        """根据板块类型动态选择表单"""
        if obj:
            # 表单映射字典
            FORM_MAPPING = {
                'banner_section': BannerSectionForm,
                'hot_games': HotGamesSectionForm,
                'latest_news': LatestNewsSectionForm,
                'features': FeaturesSectionForm,
                'categories': CategoriesSectionForm,
            }
            # 根据section_key选择对应表单
            form_class = FORM_MAPPING.get(obj.section_key, BaseConfigForm)
            if form_class:
                kwargs['form'] = form_class
        elif BaseConfigForm:
            kwargs['form'] = BaseConfigForm
        
        return super().get_form(request, obj, **kwargs)
```

### 表单设计模式

每个专属表单都遵循统一的设计模式：

```python
class XXXSectionForm(forms.ModelForm):
    """板块专属配置表单"""
    
    # 1. 定义表单字段
    config_xxx = forms.CharField(...)
    
    class Meta:
        model = HomeLayout
        fields = '__all__'
    
    # 2. __init__: 从JSON读取初始值
    def __init__(self, *args, **kwargs):
        super().__init__(*args, **kwargs)
        if self.instance and self.instance.config:
            config = self.instance.config
            self.fields['config_xxx'].initial = config.get('xxx', 'default')
    
    # 3. clean: 将表单数据组装成JSON
    def clean(self):
        cleaned_data = super().clean()
        config = {
            'xxx': cleaned_data.get('config_xxx', 'default'),
        }
        cleaned_data['config'] = config
        return cleaned_data
```
