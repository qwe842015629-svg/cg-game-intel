# 轮播图多语言翻译修复说明

## 🐛 问题描述

轮播图的文字内容（标题、描述、徽章、按钮）在切换语言时没有实时更新，始终显示初始化时的语言。

## 🔍 问题原因

在 `HomePage.vue` 中，轮播图数据 `carouselSlides` 使用的是 `ref()` 定义的静态数据：

```typescript
// ❌ 错误写法 - 静态数据，不响应语言变化
const carouselSlides = ref([
  {
    id: 1,
    title: $t('carouselTitle1'),  // 只在组件初始化时调用一次
    description: $t('carouselDesc1'),
    badge: $t('carouselBadgeHotSale'),
    // ...
  }
])
```

这种方式会在组件初始化时调用 `$t()` 函数一次，之后即使切换语言，这些文本也不会更新。

## ✅ 解决方案

将 `carouselSlides` 改为 **计算属性（computed）**，使其能够响应语言变化：

```typescript
// ✅ 正确写法 - 使用计算属性，响应语言变化
import { useLanguageStore } from '../stores/language'

// 使用语言 store
const languageStore = useLanguageStore()

// 创建响应式翻译函数
const t = (key: string) => {
  // 访问 currentLocale 以建立响应式依赖
  const _ = languageStore.currentLocale
  return languageStore.t(key as any)
}

const carouselSlides = computed(() => [
  {
    id: 1,
    title: t('carouselTitle1'),  // 每次语言切换时重新计算
    description: t('carouselDesc1'),
    badge: t('carouselBadgeHotSale'),
    primaryButton: t('startRechargeNow'),
    secondaryButton: t('viewDetails'),
    // ...
  },
  // 其他轮播项...
])
```

## 🔧 技术要点

### Vue 3 响应式原理

- **ref()**: 创建响应式引用，但值在初始化时确定
- **computed()**: 创建计算属性，依赖项变化时自动重新计算
- **useI18n()**: Vue I18n 的 Composition API，提供响应式的 `t()` 函数

### 为什么需要 computed？

当语言切换时：
1. `locale` 状态改变
2. 所有使用 `t()` 函数的**计算属性**会自动重新计算
3. 模板中引用这些计算属性的地方会自动更新

如果使用 `ref()`:
1. `locale` 状态改变
2. `ref` 的值不会自动更新（因为它在初始化时已经确定）
3. 模板不会更新

## 📊 性能影响

- **计算成本**: 极低（仅重新生成一个包含4个对象的数组）
- **触发频率**: 仅在语言切换时（用户主动操作）
- **内存占用**: 可忽略不计
- **用户体验**: 显著提升（实时翻译）
