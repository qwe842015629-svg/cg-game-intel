  ```python

          'ENGINE': 'django.db.backends.mysql',
          'NAME': config('DB_NAME', default='game_recharge'),
          'USER': config('DB_USER', default='root'),
          'PASSWORD': config('DB_PASSWORD', default=''),
          'HOST': config('DB_HOST', default='localhost'),
          'PORT': config('DB_PORT', default='3306'),
          'OPTIONS': {
              'charset': 'utf8mb4',
              'init_command': "SET sql_mode='STRICT_TRANS_TABLES'",
          },
      }
  }
  ```

## 🔧 需要用户完成的步骤

### 步骤 1: 创建 MySQL 数据库

```bash
# 登录 MySQL
mysql -u root -p

# 创建数据库
CREATE DATABASE game_recharge CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 退出
EXIT;
```

### 步骤 2: 配置数据库密码

编辑 `.env` 文件，将 `your_password` 替换为实际的 MySQL 密码：
```env
DB_PASSWORD=your_actual_password
```

### 步骤 3: 测试数据库连接

```bash
python test_mysql.py
```

**预期输出**：
```
============================================================
MySQL 数据库连接测试
============================================================

✅ 成功连接到 MySQL
   MySQL 版本: 8.0.xx
   当前数据库: game_recharge
   数据库字符集: utf8mb4
   排序规则: utf8mb4_unicode_ci
```

### 步骤 4: 执行数据库迁移

```bash
python manage.py migrate
```

### 步骤 5: 创建超级用户

```bash
python manage.py createsuperuser
```

或运行脚本：
```bash
python create_superuser.py
```

### 步骤 6: 生成测试数据

```bash
python create_test_data.py
```

### 步骤 7: 启动服务器

```bash
python manage.py runserver
```

## 🎯 优势对比

### MySQL 优势
- ✅ **并发性能更好** - 支持多用户同时访问

- ✅ **扩展性强** - 支持大数据量
- ✅ **备份恢复方便** - 成熟的备份工具

### SQLite 特点
- 适合开发和测试
- 无需单独的数据库服务器
- 配置简单
- 不适合高并发场景

## ⚠️ 重要提示

### 1. 环境变量安全
- ❌ 不要将 `.env` 文件提交到 Git
- ✅ `.gitignore` 已配置忽略 `.env`
- ✅ 使用 `.env.example` 作为模板

### 2. 密码安全
- 生产环境使用强密码
- 不要使用 root 用户
- 创建专用数据库用户

### 3. 字符集配置
- 必须使用 `utf8mb4` 字符集
- 支持完整的 Unicode（包括 Emoji）
- 避免中文乱码问题

### 4. 备份策略
```bash
# 定期备份
mysqldump -u root -p game_recharge > backup.sql

# 恢复
mysql -u root -p game_recharge < backup.sql
```

## 📞 故障排查

### 问题 1: 连接失败
```
错误: Can't connect to MySQL server
解决: 检查 MySQL 服务是否运行
命令: Get-Service MySQL*
```

### 问题 2: 认证失败
```
错误: Access denied
解决: 检查用户名和密码
```

### 问题 3: 字符集错误
```
错误: Incorrect string value
解决: 确保数据库使用 utf8mb4
```

### 问题 4: 迁移失败
```
错误: Table already exists
解决: 删除旧表或使用 --fake 参数
```
