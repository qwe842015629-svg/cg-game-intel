# 导航栏下拉菜单故障排查指南

## 问题现象
导航栏的"游戏"和"资讯"菜单悬停时没有显示下拉内容。

## ✅ 代码验证
已确认代码完全正确：
- ✅ 响应式变量已声明：`showGamesMenu`, `showNewsMenu`
- ✅ 鼠标事件已绑定：`@mouseenter`, `@mouseleave`
- ✅ 条件渲染已配置：`v-if="showGamesMenu"`
- ✅ 样式类已添加：包含 z-index、定位等

## 🔧 解决方案

### 方案 1：硬刷新浏览器（推荐）⭐

**Windows/Linux:**
```
Ctrl + Shift + R
或
Ctrl + F5
```

**Mac:**
```
Cmd + Shift + R
或
Cmd + Option + R
```

### 方案 2：清除浏览器缓存

**Chrome:**
1. 按 `F12` 打开开发者工具
2. **右键点击**刷新按钮
3. 选择"清空缓存并硬性重新加载"

**Firefox:**
1. 按 `Ctrl + Shift + Delete`
2. 勾选"缓存"
3. 点击"立即清除"

### 方案 3：无痕模式测试

打开浏览器的无痕/隐私模式：
- **Chrome**: `Ctrl + Shift + N`
- **Firefox**: `Ctrl + Shift + P`
- **Edge**: `Ctrl + Shift + N`

访问：`http://localhost:5179/`

### 方案 4：停止并重启 Vite 开发服务器

```bash
# 1. 停止当前服务器（在终端按 Ctrl+C）
# 2. 清除 node_modules/.vite 缓存
cd e:\小程序开发\游戏充值网站\frontend
rmdir /s /q node_modules\.vite

# 3. 重新启动
npm run dev
```

### 方案 5：检查浏览器控制台

1. 按 `F12` 打开开发者工具
2. 切换到 **Console** 标签
3. 查看是否有 JavaScript 错误
4. 切换到 **Network** 标签，筛选 "Navbar.vue"
5. 确认文件已重新加载（状态码 200）

## 🐛 常见问题

### Q1: 鼠标悬停没反应
**A**: 这是浏览器缓存问题，执行**硬刷新**（方案 1）

### Q2: 下拉菜单一闪而过
**A**: 检查是否鼠标移动过快，移出了菜单区域。下拉菜单需要保持鼠标在父元素或下拉内容上。

### Q3: 移动端点击没反应
**A**: 
1. 确保在移动视图模式下（宽度 < 768px）
2. 先点击汉堡菜单展开主菜单
3. 再点击"游戏"或"资讯"

### Q4: 样式不对
**A**: 
1. 确认 Tailwind CSS 已正确配置
2. 检查 `tailwind.config.js` 中是否包含自定义颜色
3. 验证 `src/assets/main.css` 中的 Tailwind 指令
