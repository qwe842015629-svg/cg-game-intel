# 🎯 Djoser 邮件激活链接修复说明

## ✅ 已完成的修复

### 问题描述
- **原问题**：邮件中的激活链接指向 `http://localhost:8000/activate/...`（后端）
- **期望结果**：激活链接应该指向 `http://localhost:5178/activate/...`（前端）

### 根本原因
Djoser 默认使用 HTTP 请求的 `HOST` 头部信息来生成邮件链接，而不是配置的 `DOMAIN`。当前端（`localhost:5178`）向后端（`localhost:8000`）发送请求时，后端看到的 HOST 是 `localhost:8000`。

---

## 🔧 修复方案

### 1. 创建自定义邮件类
**文件**：`main/email.py`

创建了 `CustomActivationEmail` 类，强制使用配置的前端域名：

```python
from djoser import email as djoser_email

class CustomActivationEmail(djoser_email.ActivationEmail):
    def get_context_data(self):
        context = super().get_context_data()
        # 强制使用正确的域名和协议
        context['protocol'] = 'http'
        context['domain'] = 'localhost:5178'
        return context
```

### 2. 更新 Djoser 配置
**文件**：`game_recharge/settings.py`

在 DJOSER 配置中添加了 `EMAIL` 配置项：

```python
DJOSER = {
    # ... 其他配置 ...
    
    # 邮件类配置（使用自定义邮件类强制使用前端域名）
    'EMAIL': {
        'activation': 'main.email.CustomActivationEmail',
    },
    
    # ... 其他配置 ...
}
```

## 📝 技术细节

### 为什么需要自定义邮件类？
Djoser 的默认行为是从 HTTP 请求中获取域名信息。由于是跨域请求（前端 5178 → 后端 8000），后端看到的域名是自己的（8000）。

通过自定义邮件类，我们可以：
1. 重写 `get_context_data()` 方法
2. 强制设置 `context['domain']` 和 `context['protocol']`
3. 确保邮件模板使用正确的域名

### 邮件模板变量
邮件模板 `templates/email/activation.html` 使用以下变量：
- `{{ protocol }}` - 协议（http 或 https）
- `{{ domain }}` - 域名和端口（localhost:5178）
- `{{ url }}` - 激活路径（activate/NQ/d31u6v-...）

最终生成的链接：`{{ protocol }}://{{ domain }}/{{ url }}`
